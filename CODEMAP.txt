# WorkSchedule 코드맵 (code-guard 필수 참조)

## 섹션 구조 (줄번호 → 역할)

### 상수/설정 (1069~1409)
DAY_START_HOUR (L1085)       하루 기준시각(03:00) — 게이지·정렬·타임바 전부 연동. 변경 시 전 로직 영향
TL_HOURS (L1093)             타임바 세로축 시간 배열 (12시 시작, 25칸)
DEFAULT_EMPLOYEES (L1100)    기본 직원 8명 (처음 실행 시 Firebase에 저장)
KR_HOLIDAYS / KR_HOLIDAY_NAMES (L1114~1173)  법정공휴일 Set + 이름 맵 (2026~2027)
SHIFT_TYPES (L1196)          근무형태 색상 시스템 (새벽/오전/피크/저녁/마감/풀타임/야간/휴무)
EMP_CAPABILITIES (L1293)     직원별 가능 역할 (수정 팝업 비활성화 판정 기준)
ROLE_LABELS / ROLE_COLORS (L1305~1306)  역할 표시명 + 색상 상수
STAFF_PRESETS (L1319~1356)   4개 매출구간 프리셋 (250/300/350/맥스) — 시간별 인원 기준
WEEKDAY_REVENUE_DEFAULTS (L1408)  요일별 기본 매출구간 (0=일~6=토)

### 고정 스케줄 (L1244~1290)
FIXED_SCHEDULES              직원별 기본 고정 스케줄 (let — 런타임 변경 가능)
getFixedScheduleForDate()    날짜+직원→고정스케줄 반환 (공휴일/요일 분기, 권연옥 조건부)
  → FIXED_SCHEDULES, isWeekendOrHoliday() 의존

### 유틸리티 (L1603~1687)

### 데이터 로드 (L1861~2001)

  3) ws_init_ver=v9 체크 → 아니면 전체 스케줄/휴무 캐시 클리어

## 공유 상태 (변경 시 연쇄 영향)

### 변경 금지 / 주의 변수
currentDate          모든 렌더 기준. setDate로 직접 변경 후 onDateChange() 필수
employees            SSE put/patch로 갱신. lsSaveEmployees 동기화 필수
daySchedule          당일 스케줄 캐시. lsSaveSchedule 동기화 필수. SSE 수신 시 덮어쓰기
weekSchedules        주간 캐시. loadWeekSchedules 완료 후 최신
shiftStatus          확정 상태. lsSaveShiftStatus + Firebase 동기화 필수
dayoffs              휴무. lsSaveDayoffs + Firebase 동기화 필수. false=수동해제(재생성 방지)
confirmedDays        날짜확정. lsSaveConfirmed + Firebase 동기화 필수
staffSettings        인원 프리셋 오버라이드. saveStaffSettings() 호출 필수
dailyRevenueMap      날짜별 매출구간. saveDailyRevenueMap() 호출 필수
FIXED_SCHEDULES      고정 스케줄 (let). 직접 변경 시 localStorage+Firebase도 갱신 필요
geminiKey            날씨/AI 공통 키. loadAiKey() 비동기 로드
hourlyWeather        날씨 데이터. renderBriefing에서 직접 참조
sseGeneration        SSE 구버전 핸들러 폐기 카운터. connectSSE() 호출마다 증가
