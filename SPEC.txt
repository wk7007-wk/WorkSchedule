# WorkSchedule 상세 스펙
# 규칙(CLAUDE.md)과 분리된 상세 구현 명세

## 직원별 능력 (가능 역할)
| 직원 | 주방 | 차배달 | 오토바이 |
|------|:----:|:------:|:--------:|
| 이원규 | O | O | O |
| 박준모 | O | O | - |
| 리 | O | - | - |
| 히오 | O | - | - |
| 권연옥 | O | - | - |
| 사아야 | O | - | - |
- 수정 팝업에서 능력 밖 역할은 비활성화(취소선)

## 고정 스케줄
| 직원 | 시간 | 역할 | 비고 |
|------|------|------|------|
| 이원규 | 17:00~03:00 | 주방+오토바이 | 매일 |
| 히오 | 17:00~03:00(금토~05, 일~02) | 주방 | 월 휴무 |
| 리 | 17:00~03:00(금토~05, 일~02) | 주방 | 화 휴무 |
| 박준모 | 평일03~11, 주말07~17 | 주방+차배달 | 일 휴무 |
| 권연옥 | 월17:30~03, 화15~24, 금17:30~03 | 주방 | 수목토일 휴무 |
| 사아야 | 월화수 17:00~22:00 | 주방 | 목금토일 휴무 |
- 자동 적용: 기존 입력 없고 휴무 아닌 경우에만
- 기존 입력 보존: 절대 덮어쓰지 않음

## 인원 기준 프리셋 (ws_staff_settings)
| 프리셋 | 6~17시 | 피크(18~19) | 야간(20~02) |
|--------|:------:|:-----------:|:-----------:|
| 250만(기본) | min1/max1 | min3/max3 | min2/max3 |
| 300만 | min1/max2 | min4/max5 | min3/max4 |
| 350만 | min1/max2 | min4/max5 | min3/max4 |
| 맥스 | min1/max2 | min5/max6 | min4/max5 |
- 오토바이/차배달: max1/min0
- 요일별 기본: 월350, 화수목250, 금300, 토300, 일350
- 커스텀: 인원 모달에서 0.5 단위 조절 → presetOverrides

## 편입 로직
1. 전원 주방 시작 (주방 공백 불가)
2. 주방 min 충족 → 초과 인력 배달 편입
3. 편입 우선: 오토바이 > 차배달
4. 편입 후 주방 min 이하 안 내려감

## 배달대행 규칙
- 바이크만 (차 안 탐), 영업 10:00~01:00
- 부족분만: daFill = max(0, 오토바이max - 실인원)
- 01~03시 (대행 마감): 배달 합산 min1/max1

## 휴무 관리
- 등록: 개별(직원+날짜) / 일괄 텍스트 입력
- 파싱: M/D콤마, 범위(~), ISO, 요일
- 매주 반복: 리=화, 히오=월, 권연옥=수목토일, 박준모=일, 사아야=목금토일
- 타임라인: 반투명 + 빨간 "휴무" 오버레이
- 해제: dayoffs[empId][dk] = false (delete 아님)
- generateAutoDayoffs: === undefined일 때만 생성

## UI 구조
1. 헤더: 날짜+◀▶+직원관리+휴무+확정/작업중+새로고침+월간
2. 브리핑 패널: 휴무자/주간누적/날씨/매출변동/근무현황
3. 탭: 타임바(메인) / 리스트 / 주간
4. 타임라인: 06:00~03:00, 직원별 근무바+역할+휴무+충돌
5. 합계행, 인원체크, 주간개요
6. 정보: 날씨+경기+물류
7. AI 스케줄링 (Gemini), 공유 (.ics 등)

## 수정 팝업
- 텍스트 select + 게이지형 바 (양방향 동기화)
- 빠른선택: 10-18, 10-20, 17-03, 18-03, 16-24, 풀
- 역할 선택: 능력 기반, 불가능=비활성화

## 날짜 스트립
- 하단 60일 (-7~+52) 가로 스크롤
- 확정=초록테두리, 부분=연초록+비율, 없음=회색
- 공휴일: KR_HOLIDAYS Set (2026-2027)
