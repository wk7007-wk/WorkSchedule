# WorkSchedule 프로젝트 규칙

## 앱 개요
매장 직원(7명) 근무표 작성/공유 앱. WebView + GitHub Pages 구조. 타임라인 게이지 + AI 스케줄링.

## 데이터 저장 (최우선 규칙)
- **localStorage = Single Source of Truth** (Firebase = 백업/동기화 전용)
- **모든 저장**: localStorage 먼저 → Firebase 백그라운드 동기화
- **모든 로드**: localStorage 즉시 렌더 → Firebase 나중에 병합 (로컬 우선)
- **새로고침/오프라인 시 데이터 유실 절대 금지**
- **Firebase 실패해도 로컬 데이터 유지** → "로컬 저장됨" 토스트
- **Firebase → localStorage 덮어쓰기 금지**: 병합 시 로컬이 항상 우선
- **clearCache 금지**: WebView clearCache(true) 사용 시 localStorage 전부 삭제됨
- **localStorage 키**:
  - 직원: `ws_employees`
  - 스케줄: `ws_sched_{yyyy-MM-dd}`
  - 휴무: `ws_dayoffs`
  - 확정상태: `ws_confirmed`

## 직원 목록
이원규, 권연옥, 리, 히오, 박준모, 전묘정, 대유

## 역할 시스템
- **3가지 역할**: 주방 / 차배달 / 오토바이
- **색상 텍스트로 표시** (이모지/그림 아이콘 지양 — 식별 어려움)
  - 주방 = #E67E22 (주황)
  - 차배달 = #4ECDC4 (민트)
  - 오토바이 = #FFD700 (노랑)
- **멀티 역할**: 동시 체크 시 각 역할 0.5명으로 계산
- **단일 역할**: 1.0명으로 계산
- **미지정**: 주방으로 카운트
- **저장 형식**: `role: "주방,차배달"` (콤마 구분)

## 고정 스케줄 (자동 입력)
| 직원 | 시간 | 역할 | 비고 |
|------|------|------|------|
| 이원규 | 06:00~20:00 | 주방+오토바이 | 매일 고정 (새벽6시~저녁8시) |
| 히오 | 18:00~03:00 | 주방 | 매일 고정 |
| 리 | 18:00~03:00 | 주방 | 히오와 동일, 매일 고정 |
| 박준모 | 17:00~03:00 | 주방+차배달 | 매일 고정, 휴무는 나중 지정 |
| 전묘정 | 17:00~03:00 | 주방 | 매일 고정, 휴무는 나중 지정 |
| 대유 | - | - | 변칙 — 수동 입력 (고정 없음) |
| 권연옥 | - | - | 미지정 (추후 확인) |
- **자동 적용 조건**: 해당 날짜에 기존 입력이 없고, 휴무가 아닌 경우에만
- **기존 입력 보존**: 이미 입력된 스케줄은 절대 덮어쓰지 않음

## 충돌 규칙 (핵심)
### 역할별 동시 인원 제한
| 역할 | 최대 | 초과 시 |
|------|------|---------|
| 차배달 | **1명** | 1명 주방편입 → 빨간음영 |
| 오토바이 | **1명** | 1명 주방편입 → 빨간음영 |
| 주방 (피크 17~20시) | **4명** | 초과분 빨간음영 |
| 주방 (그 외 시간) | **3명** | 초과분 빨간음영 |

### 주방 최소 인원
| 시간대 | 최소 |
|--------|------|
| 17:00~03:00 | **2명** |
| 그 외 | **1명** |

### 충돌 표시 방법
- **초과 직원**: 시프트 블록 **빨간 음영** + 빨간 좌측 바 + 경고 텍스트
- **시간대별 인원 체크**: 초록=충족, 빨강=초과, 주황=부족

## 보정 시스템 (핵심 워크플로우)
### 상태 흐름
- **보정 (auto)**: 고정근무자 자동 적용값. 아직 확인/수정 전 기본 상태
- **확정 (confirmed)**: 수동 저장 시 자동 전환. 또는 확 버튼 터치. 패턴 기록됨
- **미정 (pending)**: 불확실한 경우. 미 버튼 터치

### 보정→확정 워크플로우
1. 매일 시작 시 고정근무자 자동 배치 → "보정" 상태
2. 관리자가 확인 → 그대로면 "확" 버튼, 수정 필요하면 수정 후 저장 → 자동 "확정"
3. 대유 등 변칙근무자 → 직접 입력 or AI 추천 적용
4. 전체 확인 완료 → "전체확정" 버튼

### AI 패턴 학습
- 확정 시 `ws_patterns` + Firebase `/workschedule/patterns/` 에 패턴 기록
- 키: `직원명_요일번호` (예: "대유_2" = 대유의 화요일)
- 확정 2회 이상 → 미입력 직원에 "AI추천" 시간대 표시 + 적용 버튼
- 추천 정확도: 확정 비율 기반 confidence %

### 보정 진행률 게이지
- 리스트 상단에 색상 게이지 바 (확정=초록/보정=노랑/미정=주황/미입력=빨강)
- 전체확정/전체보정 일괄 버튼

### localStorage 키
- `ws_shift_status`: 직원별 일별 보정 상태
- `ws_patterns`: AI 학습용 확정 패턴
- `ws_confirmed`: 일별 확정 상태
- `ws_employees`, `ws_sched_{yyyy-MM-dd}`, `ws_dayoffs`

## 일별 확정/작업중 상태
- 헤더에 **"확정"/"작업중" 토글 버튼**
- **확정**: 초록 테두리+배경 → 근무표 완성 상태
- **작업중**: 주황 텍스트 → 아직 편집 중 (기본값)
- localStorage `ws_confirmed` 키에 저장

## 배달대행 (가상 멤버)
- **영업시간**: 10:00 ~ 01:00 (새벽 1시)
- 배달자(차+오토바이) 공백 시간대에 자동으로 배달 파트 충족
- 배달 인원 부족 + 영업시간 내 → "대행"으로 표시

## 인원 규칙 (시간대별)
- 매시간 **주방 1명 + 배달(차 or 오토바이) 1명** 필요
- 주방 최소/최대: 위 충돌 규칙 참조
- 차배달/오토바이 최대: 각 1명
- 물류: 16:30 이후 인원 체크
- **충족 색상**: 초록=OK, 빨강=초과, 주황=부족

## 앱 성격 (보고서/모니터링 중심)
- **보고서 형태 우선**: 근무자에게 전달 시 보기 좋은 형태 유지
- **수정보다 모니터링**: 직접 입력/수정은 게이지+팝업, 대부분 AI가 담당
- **여러 형태의 보고서**: 세로/달력/가로/리스트 탭으로 다양한 뷰 제공
- **사진 공유용 가시성**: 직원명+시간 크고 명확, 스크린샷으로 전달 가능

## 레이아웃 탭 시스템
- **4개 탭**: 리스트(기본/메인) / 세로 / 가로 / 달력
- **리스트 탭 (메인)**: 보정 기반 직원 리스트. 출퇴근 텍스트 + 타임라인 게이지 + 확/미 상태 버튼. 보정 진행률 게이지. AI 패턴 추천.
- **세로 탭**: 세로 타임라인 게이지 (06:00~03:00, 직원별 열), 온도 표기 삭제(가시성 저해)
- **달력 탭**: 월간 그리드 (미래만 표시, 과거 비활성), 셀 터치 → 해당 날짜 리스트탭 전환
- **가로 탭**: 직원별 수평 바 차트
- **리스트 탭**: 이름/시간/역할 목록 형식, 근무/휴무/미입력 구분
- **모든 탭에서 터치 → 팝업 수정 가능** (드래그 입력 없음, 터치=팝업)
- **주간개요**: 미래 날짜만 강조 (과거 반투명)

## 수정 팝업 (듀얼 방식)
- **텍스트 선택**: select 드롭다운으로 시작/종료 시간 선택
- **게이지형**: 수평 바 터치/드래그로 시간 조절
- **양방향 동기화**: select 변경 ↔ 게이지 자동 연동
- **빠른선택**: 프리셋 버튼 (10-18, 10-20, 17-03, 18-03, 16-24, 풀)
- **역할 선택**: 직원 능력 기반, 불가능한 역할은 비활성화
- **재배치 버튼**: 고정근무자 클리어 후 재배치

## UI/레이아웃 규칙
- **기능 축소/숨김 금지**: display:none으로 섹션 숨기지 말 것
- **타임라인**: 70vh 높이, 스크롤 가능 (overflow-y:auto)
- **헤더**: sticky top, compact (불필요 아이콘 최소화)
- **시간 가독성**: 폰트 .7rem 이상, 색상 #9090A8 이상 밝기
- **body**: 자연스러운 스크롤 (overflow:hidden 금지)
- **시프트 블록**: 직원명(.75rem bold 흰색) + 시간 + 역할(색상텍스트) + 시간수
  - 배경 55% 불투명 + 좌측 3px 컬러바
  - 충돌 시: 빨간 음영 + 빨간 좌측 바
- **이모지/그림 아이콘 지양**: 색상 텍스트로 역할 식별
- **세로 타임라인 시간열**: 32px (온도 삭제로 축소)
- **스포츠/이슈**: 경기 없는 날엔 표기 X

## 휴무 관리
- **휴무 버튼**: 헤더에 빨간 "휴무" 버튼
- **등록 방식**: 개별(직원+날짜) / 일괄 텍스트 입력
- **일괄 입력 파싱**:
  - `이원규 2/25, 2/26` — M/D 콤마
  - `권연옥 3/1~3/3` — 범위
  - `리 2025-03-05` — ISO
  - `히오 화,목` — 이번주 요일
- **매주 반복 휴무 (자동)**:
  - 리: 매주 월요일
  - 히오: 매주 화요일
- **기간 휴무**: 권연옥 2026-02-24 ~ 2026-03-08
- **타임라인 표시**: 휴무 직원 = 반투명 + 빨간 "휴무" 오버레이
- **Firebase**: `/workschedule/dayoffs/{empId}/{yyyy-MM-dd}`
- **localStorage**: `ws_dayoffs`
